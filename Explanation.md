# Project Explanation & Code-to-Feature Mapping

This document provides a comprehensive overview of the `mrares-portfolio` project's structure, highlighting how the source code maps to the features, animations, and elements visible on the website. This guide is intended to facilitate future development and maintenance.

## Tech Stack Overview

- **Framework:** React 18 with TypeScript, built using Vite.
- **Styling:** Tailwind CSS combined with `shadcn/ui` (built on Radix UI primitives).
- **Routing:** React Router v6 (`react-router-dom`).
- **State/Data Management:** React Query (`@tanstack/react-query`).
- **Animations:** Custom React hooks (`useScrollReveal`, `useParallax`), Tailwind CSS keyframes, and `tailwindcss-animate`.
- **Icons:** Lucide React (`lucide-react`).

## Application Entry & Routing

- **`src/main.tsx`**: The main entry point that mounts the React application to the DOM.
- **`src/App.tsx`**: The root component where global providers and routing are defined.
  - Wraps the app in `QueryClientProvider`, `LanguageProvider`, and `TooltipProvider`.
  - Includes global UI elements: `ScrollProgress`, `ScrollToTop`, `Toaster` (shadcn UI), and `Sonner` (for toast notifications).
  - Defines the main routes:
    - `/` -> `Index` (Main Portfolio Page)
    - `/project/insaight` -> `InsaightProject` (Details for Insaight)
    - `/project/noli` -> `NoliProject` (Details for Noli)
    - `*` -> `NotFound` (404 Page)

## Pages (`src/pages/`)

These files represent the main views of the application.

- **`Index.tsx`**: The primary landing page. It acts as a container that brings together all the individual sections (Hero, Experience, Projects, etc.).
- **`InsaightProject.tsx`**: A dedicated case study/details page for the "Insaight" project.
- **`NoliProject.tsx`**: A dedicated case study/details page for the "Noli" project.
- **`NotFound.tsx`**: The fallback 404 page displayed when a user navigates to an invalid URL.

## Features & Components (`src/components/`)

The application is modularized into feature-specific components.

### Layout & Navigation
- **`Header.tsx`**: The top navigation bar, containing branding, navigation links (`NavLink.tsx`), and potentially language/theme toggles.
- **`Footer.tsx`**: The bottom section of the site containing copyright info, social links, and secondary navigation.

### Landing Page Sections
These components are sequentially rendered on the `Index.tsx` page to form the single-page portfolio layout:
- **`HeroSection.tsx`**: The prominent introductory section at the top of the page, usually containing the main headline, introductory text, and a primary call-to-action (CTA).
- **`ExperienceSection.tsx`**: Displays professional work history and timeline.
- **`EducationSection.tsx`**: Showcases academic background and degrees.
- **`SkillsSection.tsx`**: Highlights technical skills, tools, and languages (often displayed as badges or categorized lists).
- **`CertificationsSection.tsx`**: Lists professional certifications and achievements.
- **`FeaturedProjectsSection.tsx`**: A showcase grid or list of key projects, containing links that navigate to the detailed project pages (e.g., `/project/insaight`).
- **`ContactSection.tsx`**: The final section, typically containing a contact form or direct links (email, LinkedIn) to get in touch.

### Base UI Components (`src/components/ui/`)
Contains all the reusable primitive components generated by `shadcn/ui`. Some notable elements used across the site:
- **Buttons, Cards, Badges:** Used extensively in sections like `SkillsSection` and `FeaturedProjectsSection`.
- **Carousel:** Used for sliding through images or projects.
- **Accordions/Dialogs/Sheets/Sidebars:** Provide interactive layout and disclosure widgets.

## Animations and Interactions

The portfolio uses a mix of Tailwind config and custom React hooks to handle animations:

1. **Custom Hooks (`src/hooks/`)**:
   - **`useScrollReveal.ts`**: Handles the "fade-in-up" animations when elements scroll into the viewport. This gives the smooth scrolling experience where content appears dynamically.
   - **`useParallax.ts`**: Creates parallax scrolling effects (typically used for background elements or images shifting slightly at a different speed than the main scroll).
   - **`use-mobile.tsx`**: A utility hook to detect screen sizes and adjust interactive elements or animations for mobile devices.

2. **Tailwind Configurations (`tailwind.config.ts`)**:
   - Uses the plugin `tailwindcss-animate`.
   - Defines custom keyframes such as `fade-in-up`, `accordion-down`, and `accordion-up` which are applied as utility classes (e.g., `animate-fade-in-up`) on elements to trigger CSS animations.

3. **Visual Effects Components**:
   - **`CustomCursor.tsx`**: Replaces the default browser cursor with a custom, stylized cursor that can interact with hovered elements.
   - **`BackgroundEffects.tsx`**: Manages dynamic, animated, or styled background elements that sit behind the main content to enhance visual depth.
   - **`ScrollProgress.tsx`**: A visual indicator (bar fixed at the top of the screen) that fills up as the user scrolls down the page.
   - **`ScrollToTop.tsx`**: A utility component that scrolls the window back to the top when routes change.

## Contexts & Utilities

- **Translations (`src/lib/translations.ts`)**: This file contains all the string dictionaries (English, Spanish, etc.) used across the site.
- **Language Context (`src/contexts/LanguageContext.tsx`)**: Manages the currently selected language state and provides the translation strings from `translations.ts` to all child components, ensuring the portfolio is multilingual.
- **Utility Functions (`src/lib/utils.ts`)**: Typically contains helper functions like `cn()` (using `clsx` and `tailwind-merge`) for conditional Tailwind CSS class merging.

## Future Development Guidelines

1. **Adding a New Project**:
   - Create a new component in `src/pages/` (e.g., `NewProject.tsx`).
   - Add a new route in `src/App.tsx` (e.g., `<Route path="/project/new" element={<NewProject />} />`).
   - Update `src/components/FeaturedProjectsSection.tsx` to include a card linking to the new project route.
2. **Adding a New Section to the Homepage**:
   - Create the section component in `src/components/` (e.g., `TestimonialsSection.tsx`).
   - Import and render it in `src/pages/Index.tsx` at the desired location.
3. **Updating Translations**:
   - Go to `src/lib/translations.ts` to add or update the text content for the different languages.
4. **Adding New Animations**:
   - Add new keyframes in `tailwind.config.ts` or leverage existing logic in `src/hooks/useScrollReveal.ts` on your new components.
